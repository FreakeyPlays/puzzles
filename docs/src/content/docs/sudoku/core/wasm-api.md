All public functions live in `sudoku-wasm/src/lib.rs` and are annotated with `#[wasm_bindgen]`.

## Board Format

All functions that accept or return a board use an **81-character string**:

- Characters `'1'`–`'9'` represent filled cells
- Character `'0'` represents an empty cell
- Index formula: `row * 9 + col` (0-based)

```
"530070000600195000098000060800060003400803001700020006060000280000419005000080079"
 ^                                                                               ^
 index 0 (row 0, col 0)                                           index 80 (row 8, col 8)
```

## Difficulty

```rust
// Valid values: "easy" | "medium" | "hard" | "extreme"
// Default when None: "medium"
type Difficulty = Option<String>;
```

## Functions

### `generate`

Generates a new puzzle with a guaranteed unique solution.

```rust
#[wasm_bindgen]
pub fn generate(difficulty: Option<String>, seed: Option<u32>) -> Result<JsValue, JsError>
```

If `seed` is `None`, a random seed is generated internally and returned in the result — every generated puzzle is reproducible.

**Returns** (`GenerateResult`):

```rust
#[derive(Serialize)]
pub struct GenerateResult {
    pub puzzle: String,       // 81 chars — the unsolved board
    pub solution: String,     // 81 chars — the unique solution
    pub difficulty: String,   // actual difficulty that was applied
    pub seed: u32,            // seed used — always set, even if not provided as input
}
```

**Errors:** Returns `JsError` if `difficulty` is not one of the valid values.

---

### `solve`

Computes the solution for a given board. Returns `null` if the board is unsolvable (e.g. the player introduced a contradiction).

```rust
#[wasm_bindgen]
pub fn solve(board: &str) -> Result<Option<String>, JsError>
```

**Returns:** `String | null` — the completed 81-character board, or `null` if no solution exists.

**Errors:** Returns `JsError` if `board` is not exactly 81 characters or contains invalid characters.

---

### `validate`

Checks the current board state for rule violations (duplicate values in any row, column, or 3×3 box). Does **not** compare against the known solution.

```rust
#[wasm_bindgen]
pub fn validate(board: &str) -> Result<JsValue, JsError>
```

**Returns** (`ValidateResult`):

```rust
#[derive(Serialize)]
pub struct ValidateResult {
    pub valid: bool,           // true if no rule violations exist
    pub solved: bool,          // true if valid AND all 81 cells are filled
    pub conflicts: Vec<u8>,    // indices (0–80) of cells involved in a conflict; empty if valid
}
```

**Errors:** Returns `JsError` if `board` is not exactly 81 characters or contains invalid characters.

---

### `hint`

Returns the easiest move currently available — the one requiring the lowest-complexity solving technique. Returns `null` if the board is already solved or in an invalid state.

```rust
#[wasm_bindgen]
pub fn hint(board: &str) -> Result<Option<JsValue>, JsError>
```

**Returns** (`HintResult | null`):

```rust
#[derive(Serialize)]
pub struct HintResult {
    pub index: u8,         // cell index (0–80)
    pub value: u8,         // value to place (1–9)
    pub technique: String, // e.g. "naked_single", "hidden_single", "naked_pair", …
}
```

**Errors:** Returns `JsError` if `board` is not exactly 81 characters or contains invalid characters.

---

## Error Handling

All functions return `Result<_, JsError>` instead of panicking on invalid input. `wasm-bindgen` maps a returned `Err(JsError)` to a thrown JavaScript exception, which the Web Worker's `try/catch` catches and forwards as a `{ type: 'error' }` message to the main thread.

Callers should only ever encounter errors if they pass malformed boards — boards generated by `generate` and edited only through the frontend are always valid inputs.
